Point cle du sujet: il faut 14 points minimum pour valider le projet.

Important: les modules incomplets ou non fonctionnels valent 0 point pendant l'evaluation.

Bonus: au-dela des 14 points, le sujet limite le bonus pris en compte a 5 points maximum. Donc l'objectif reel comptabilisable pendant la review est 19 points, pas 20/21.

Ce document liste quand meme tous les modules du sujet, meme ceux qui ne sont pas encore faits, pour servir de feuille de route et de support README/evaluation.

Lecture des statuts

  • Implemente: le repo contient deja des preuves serieuses pour soutenir ce module.
  • Partiel: il y a une base, mais il manque de la finition, de la doc, ou certains criteres du sujet.
  • A faire: pas encore implemente ou pas assez present pour etre revendique.
  • Bloque / dependance: module impossible a revendiquer tant qu'un prerequis du sujet n'existe pas.

Modules avec forte base dans le code actuel

  • Framework frontend + backend: Next.js cote front, Express cote back.
  • Temps reel: Socket.io attache au serveur HTTP.
  • Interaction utilisateurs: profils, amis, messagerie directe.
  • API publique: cle API, rate limit, Swagger, endpoints CRUD sur les posts.
  • ORM: Prisma.
  • Notifications: endpoints et systeme de lecture/suppression.
  • Upload de fichiers: images pour posts et profils avec validations.
  • i18n + RTL: au moins 4 langues visibles dans le code, dont l'arabe.
  • User management standard: profil, avatar, edition, amis, page profil.
  • OAuth et 2FA: routes et logique dediees presentes.

Strategie conseillee pour la soutenance

Objectif securise: annoncer d'abord les modules bien prouvables dans le code, puis ajouter un petit nombre de modules partiels seulement si vous avez le temps de les finir et de les documenter.

  • Socle deja defendable: 17 points environ.
  • Modules a fiabiliser pour viser 19: navigateurs supplementaires, dashboard analytics utilisateur, moderation IA, design system ou recherche avancee.
  • Ne pas promettre: modules de jeu, devops lourd, blockchain, accessibilite WCAG complete, PWA, organisations, permissions avancees, s'ils ne sont pas reellement demonstrables.

Points a ajouter ensuite dans le README

  • Liste finale des modules reellement revendiques.
  • Justification de chaque module choisi.
  • Preuve concrete d'implementation: routes, pages, composants, captures, scenarios de demo.
  • Qui a fait quoi dans l'equipe.
